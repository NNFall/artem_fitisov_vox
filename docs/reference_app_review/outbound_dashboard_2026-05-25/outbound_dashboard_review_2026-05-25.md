# Отчет по проверке референсного dashboard

Дата проверки: 25.05.2026, 17:18-17:23 по Самаре.  
URL: `https://outbound-calling-dashboard-671587661095.europe-west2.run.app/`

## Короткий вывод

Эта ссылка открывается без Google-авторизации. Это уже рабочий визуальный мокап панели управления обзвоном.

Проверены:

- desktop-экран dashboard;
- mobile-экран dashboard;
- клики по 4 метрикам;
- клики по 2 кампаниям;
- детальный экран кампании;
- кнопки `Назад`, `Приостановить`, `Контакты`, `Задачи`, `Звонки`, `События`;
- попытка состояния `Приостановить -> Запустить`;
- мобильные вкладки детального экрана.

Ключевой результат: визуальная идея понятная, но интерфейс пока больше похож на статический прототип. Есть кликабельные карточки кампаний, но они не размечены как кнопки на главном экране; кнопка `Приостановить` визуально не меняет состояние; на мобильной версии детальная таблица ломается и текст перекрывается.

Сырые данные проверки: [inspection-data.json](inspection-data.json).

## Главный экран

На главном экране отображается:

- заголовок `Панель управления обзвоном`;

Технически на начальном экране не найдено ни одного стандартного интерактивного элемента:

- `button`: 0;
- `a`: 0;
- `input/select/textarea`: 0;
- `role="button"`: 0;
- `tabindex`: 0.

При этом обе карточки кампаний реагируют на клик мышью и открывают детальный экран. Значит, кликабельность реализована несемантическим способом, без доступности с клавиатуры.

## Вкладки

### Контакты

Есть таблица:

- имя;
- телефон;
- компания;
- статус;
- город.

Контакты выглядят как тестовая база: Артем, Иван, Мария, Олег, Анна. Для desktop читается нормально.

### Задачи

Таблица показывает:

- ID задачи;
- статус;
- попытки;
- последний статус.

Проблема: внутренние статусы остаются на английском: `completed`, `failed`, `in_progress`, `pending`. Для финального русского интерфейса это лучше заменить на `Завершено`, `Ошибка`, `В работе`, `Ожидает`.

### Звонки

Таблица показывает:

- клиент;
- итог;
- длительность;
- саммари.

Критичный пробел для нашего будущего продукта: нет перехода в результат звонка, нет аудио, нет записи разговора, нет подробной карточки вызова, нет расшифровки. Пока вкладка работает только как краткая таблица.

### События

Лента событий показывает:

- начало дозвона;
- клиент поднял трубку;
- звонок завершен;
- автоответчик;
- время события справа.

Это направление хорошее: как раз такой live/status слой нужен для диагностики обзвона.

## UX/UI-оценка

### Что работает

- Общая структура понятная: сначала обзор, затем кампания, затем вкладки.
- Темная админ-панель визуально подходит под мониторинг и операционную работу.
- Лента событий хороша как идея: по ней можно понимать, где именно застрял звонок.
- Есть минимум нужных сущностей: кампании, контакты, задачи, звонки, события.

### Основные проблемы

**P1: Кликабельные кампании не размечены как интерактивные элементы.**  
На главном экране карточки открываются мышью, но не являются `button`, `a`, не имеют `role`, `tabindex`, фокуса и клавиатурного доступа. Это ломает доступность и усложняет автотесты.

**P1: `Приостановить` не меняет состояние.**  
После клика нет явного результата. Для админ-панели это опасно: пользователь не понимает, поставлена ли кампания на паузу. Для уже paused-кампании кнопка также должна быть другой.

**P1: Mobile-детали кампании не готовы.**  
Главный экран на мобильном приемлемый, но детальная страница ломается. Таблицы нужно превращать в компактные карточки или отдельные строки с раскрытием.

**P2: Нет детального результата звонка.**  
Вкладка `Звонки` показывает только краткую таблицу. Для нашей системы нужен клик в звонок: запись, итог, summary, расшифровка, статусы, стоимость, технические события.

**P2: Часть статусов на английском.**  
`completed`, `failed`, `in_progress`, `pending` выглядят как внутренние значения базы, а не как интерфейс для пользователя.

**P2: Заголовок страницы в браузере не настроен.**  
Сейчас title: `My Google AI Studio App`. Для референса это не критично, но в нашем проекте должен быть нормальный title.

**P3: Визуал пока слишком базовый для “готовой” админки.**  
Темный фон, одинаковые плашки, минимальная палитра, слабая иерархия внутри таблиц. Как каркас подходит, но надо добавить более сильные состояния, явные affordance-эффекты, hover/focus, карточку звонка и аккуратнее отработать плотность.

## Что стоит забрать в нашу версию

- Логику `обзор -> список кампаний -> детали кампании -> вкладки`.
- Отдельную вкладку событий, она полезна для диагностики звонков.
- Быстрые метрики кампании сверху.
- Разделение `Контакты / Задачи / Звонки / События`.
- Темную тему можно использовать, но сделать ее спокойнее и богаче по структуре.

## Что обязательно доработать в нашей реализации

- Карточки кампаний должны быть настоящими интерактивными элементами.
- У каждой кнопки должны быть hover, focus, active, disabled/loading-состояния.
- `Старт/Пауза` должны менять состояние в интерфейсе сразу.
- Для звонка нужна отдельная детальная карточка/страница.
- На мобильном таблицы нужно показывать карточками, а не сжимать колонки.
- Все статусы должны быть на русском.
- Нужно добавить пустые состояния, ошибки, загрузку, подтверждение действий.
- Нужно предусмотреть аудиоплеер записи разговора и блок summary.

## Итог

Референс полезен как быстрый каркас информационной архитектуры, но не как готовый визуальный и UX-стандарт. Для нашей веб-версии стоит брать структуру и набор сущностей, но дизайн, адаптивность, доступность, интерактивность и детальную страницу звонка надо проектировать заново и качественнее.
