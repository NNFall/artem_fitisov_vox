# План проекта исходящего AI-обзвона

## Этап 1. Ручной тест исходящего звонка

- отдельный сценарий Voximplant для исходящего PSTN-звонка;
- номера задаются прямо в коде;
- Gemini Live подключается после ответа клиента;
- запись разговора сохраняется средствами Voximplant;
- результат проверяется по логам.

Готовый файл: `scenarios/outbound_gemini_test.js`.

## Этап 2. Управление с backend

- backend в Docker;
- endpoint для запуска одиночного звонка;
- endpoint для загрузки списка номеров;
- таблицы для контактов, задач обзвона, попыток, записей, итогов и стоимости;
- запуск Voximplant сценария через Management API;
- прием webhooks/status callbacks от Voximplant.

Первая backend-итерация добавлена в `backend/`: FastAPI, SQLite, прием финального payload, скачивание записей, Telegram/Google Sheets delivery.

## Этап 3. Telegram-управление

- команда "позвони номеру";
- команда загрузки базы;
- просмотр статуса обзвона;
- получение итогов и ссылок на записи;
- ручная остановка кампании.

## Этап 4. Отчеты и аналитика

- итог разговора;
- статус: дозвонились, не ответил, отказ, интерес, перезвонить;
- краткая суммаризация;
- стоимость звонка;
- фильтры по кампании, дате и статусу.

## Важная граница ответственности

Voximplant-сценарий должен оставаться коротким: звонок, медиа, AI, запись, отправка результата. Очереди, база, ретраи, Telegram и отчеты должны жить на backend.
